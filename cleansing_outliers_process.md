# 数据清洁流程

## Step 0 - cleansing_data_dbcs_handling_step0.py

**目的**: 将日期列中的全角字符转换为半角字符，统一格式

此步骤在其他所有清洁步骤之前执行，确保后续步骤使用统一的半角字符格式。

### 全角转半角映射
| 全角字符 | 半角字符 | 说明 |
|----------|----------|------|
| `，` | `,` | 全角逗号 → 半角逗号 |
| `、` | `,` | 中文逗号 → 半角逗号 |
| `～` | `~` | 全角波浪号 → 半角波浪号 |
| `——` | `-` | 中文破折号 → 半角短横线 |
| `—` | `-` | 中文长破折号 → 半角短横线 |
| `（` | `(` | 全角左括号 → 半角左括号 |
| `）` | `)` | 全角右括号 → 半角右括号 |
| `　` | ` ` | 全角空格 → 半角空格 |
| `·` | `,` | 中文点号 → 半角逗号 |

### 行为
- **--dry-run**: 预览更新结果，导出到 HTML
- **无参数**: 执行更新，导出到 HTML

### 执行结果
- 更新前数据库记录: 256,063
- 更新后数据库记录: 256,063（记录数不变，日期值被更新）
- 实际更新: **1,640** 条

## Step 1 - cleansing_outliers_step1.py

**目的**: 识别并删除异常日期记录

### 异常日期判定条件
- `日期` 为 NULL 或空
- `日期` 包含中文冒号 (`：`)
- `日期` 包含 `月` 字
- `日期` 包含 `加班` 或 `半天`

| 类别 | 定义 |
|------|------|
| `real_outliers` | 关键字段(工序全名、工序、计件数量、系数、定额、金额、备注、代码)全为0或空 |
| `possible_outliers` | 至少有一个关键字段有有效数据 |

### 行为
- **--dry-run**: 仅导出到 Excel，不执行删除
- **无参数**: 仅删除 `real_outliers`，保留 `possible_outliers`

### 输出
- Excel 文件 `outliers_to_be_deleted.xlsx`，包含两个 sheet
- 删除前后的数据库记录总数

## Step 4 - cleansing_outliers_step4.py

**目的**: 将郁俊海的月份日期记录转换为该月份的第一个工作日

### 问题发现
在审查 Step 1 遗留的 412 条异常日期记录时，发现其中 **411 条** 都是郁俊海（2025年数据）的月份记录：
- `4月`, `5月`, `6月` (numeric month format)
- `三月` (Chinese month format)
- `7月份` (with 份 suffix)

| 日期模式 | 示例 | 数量 |
|----------|------|------|
| 三月 | March (Chinese) | 101 |
| 4月 | April | 75 |
| 5月 | May | 97 |
| 6月 | June | 22 |
| 7月份 | July | 116 |

**总计**: 411 条记录

### 转换结果（按文件）
| 文件名 | 原日期 | 新日期(天) | 数量 |
|--------|--------|-----------|------|
| 202503.xls | 三月 | 3 | 101 |
| 202504.xls | 4月 | 1 | 75 |
| 202505.xls | 5月 | 6 | 97 |
| 202506.xls | 6月 | 3 | 22 |
| 202507.xls | 7月份 | 1 | 116 |

### 转换逻辑
对于每条月份记录，转换为该月份的**第一个工作日的"日"（天数字符串）**：
- **工作日定义**: 周一至周五，且不是中国法定节假日
- **节假日判断**: 使用 `holidays` 模块获取中国法定节假日
- **规则**: 从每月1日开始，找到第一个 weekday（非周末、非节假日），返回其"日"数字
- **注意**: 仅返回天数字（如"3"），不返回完整日期。完整日期解析将在后续步骤进行。

### 2025年节假日列表（用于工作日计算）
| 日期 | 节日 |
|------|------|
| 2025-01-28 ~ 2025-02-04 | 春节 |
| 2025-04-04 | 清明节 |
| 2025-05-01 ~ 2025-05-02 | 劳动节 |
| 2025-05-05 | 端午节 |
| 2025-06-02 | 端午节(调休) |
| 2025-10-01 ~ 2025-10-03 | 国庆节 |
| 2025-10-06 | 中秋节 |

### 转换示例
| 原日期 | 新日期(天) | 说明 |
|--------|-----------|------|
| 三月 | 3 | 3月1日是周六，找到第一个工作日3月3日(周一)，返回"3" |
| 4月 | 1 | 4月1日是周二，非节假日，返回"1" |
| 5月 | 6 | 5月1日是周四(劳动节)，5月2日周五调休，第一个工作日是5月6日(周二)，返回"6" |
| 6月 | 3 | 6月1日是周日(端午节)，第一个工作日是6月3日(周二)，返回"3" |
| 7月份 | 1 | 7月1日是周二，非节假日，返回"1" |

### 行为
- **--dry-run**: 预览更新结果，导出到 HTML
- **无参数**: 执行更新，导出到 HTML

### 执行结果
- 更新前数据库记录: 256,063
- 更新后数据库记录: 256,063（记录数不变，日期值被更新）
- 实际更新: 411 条

## Step 5 - cleansing_date_handling_step5.py

**目的**: 将各种日期模式统一转换为逗号分隔的列表格式

### 处理模式
| Case | 模式 | 示例 | 转换结果 |
|------|------|------|----------|
| Case 2 | 中文逗号分隔 (`、`) | `1、2` | `1,2` |
| Case 2 | 英文逗号分隔 (`,`) | `1,2` | `1,2` |
| Case 3 | 点号分隔 (`．`) | `1.2` | `1,2` |
| Case 4 | 波浪号范围 (`～`) | `1～5` | `1,2,3,4,5` |
| Case 5 | 短横线范围 (`-`) | `1-5` | `1,2,3,4,5` |

### 特殊处理
- **点号+波浪号组合** (如 `15.12.2～4`): 识别为 YYYY.M.D～D 格式，展开为 `15.12.2,15.12.3,15.12.4`
- **中文逗号+短横线组合** (如 `1、2、6-10`): 先将中文逗号替换为英文逗号，再处理短横线范围，展开为 `1,2,6,7,8,9,10`

### 复杂模式 (Step 6 处理)
- 同时包含多种分隔符且不符合上述规则
- 3段式点号日期 (如 `14.6.3`) 需要文件名上下文解析完整日期
- 复杂示例: `18.1.2、3`, `2017.9.1、2`, `1、2、6-10` (有点号时)

### 行为
- **--dry-run**: 预览更新结果，导出到 HTML
- **无参数**: 执行更新，导出到 HTML

### 执行结果
- 更新前数据库记录: 256,063
- 更新后数据库记录: 256,063（记录数不变，日期值被更新）
- 实际更新: **39,223** 条
